# /security-audit

Diff-aware ревью безопасности изменений в **mission-recorder**. Анализируй **изменения**, а не весь репозиторий.

## Шаги

1. `git diff main...HEAD` — что изменилось; `git log main..HEAD` — намерение.
2. Для каждого изменения пройди таблицу угроз.
3. Сомнительное поведение библиотеки проверяй по её исходникам/официальной документации, не по памяти.
4. Репортуй только находки с **конкретным путём эксплуатации** (кто, каким вводом, что получает).

## Таблица угроз

| Категория | На что смотреть |
|---|---|
| Injection | Конкатенация ввода в SQL/command/path; `ProcessBuilder`/`Runtime.exec` со склейкой |
| AuthN/AuthZ | Маршрут без аутентификации; IDOR — объект по id без проверки владельца; hardcoded creds; слабый хэш паролей |
| Crypto | Своя крипта; статичные IV/соли; `Random` вместо `SecureRandom`; ECB; `==` для секретов (timing) |
| Secrets | Ключи/токены в коде; `.env`/`local.properties`/keystore в git; секреты в `BuildConfig`/логах |
| Serialization | Polymorphic-десериализация недоверенного ввода; Jackson default typing; `ObjectInputStream` |
| Network | Отключённый TLS; `anyHost()` CORS; cleartext; SSRF по пользовательскому URL без allowlist |
| Конкуренция | TOCTOU; гонки на общем мутабельном состоянии/файле/соединении |
| Logging | PII/токены/`Authorization`/тела запросов в логах прода |

## Severity и вердикт

- **Critical** — RCE, утечка секретов прода, обход аутентификации.
- **High** — IDOR/чужие данные, инъекция с реальным вводом, отключённый TLS.
- **Medium** — слабая крипта/хэш, недостаточная валидация на доверенном пути.
- **Low/Info** — глубина защиты, утечка в debug-логах.

По каждой находке: файл:строка → категория → severity → путь эксплуатации → рекомендация. В конце — вердикт `clean | needs-fixes | block`. Critical/High блокируют merge.

Не выдумывай уязвимости ради объёма; нет пути эксплуатации — нет находки.
