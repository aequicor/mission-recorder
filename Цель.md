# Цель Mission Recorder

## Краткое описание

Mission Recorder - бесплатный, открытый и локальный кроссплатформенный рекордер экрана на Kotlin/KMP. По пользовательским возможностям он должен закрывать сценарии уровня Bandicam: запись экрана, выбранного монитора, области, конкретного приложения, микрофона, системного звука, фонового replay buffer и экспорт кадров. При этом продукт должен оставаться прозрачным, безопасным, расширяемым и не требовать облака или подписки.

## Детальное описание

Mission Recorder создается как инструмент для разработчиков, команд и обычных пользователей, которым нужна быстрая и контролируемая запись экрана. Основной сценарий: пользователь выбирает источник записи, настраивает видео и звук, запускает запись из CLI или GUI, получает локальный файл и при необходимости экспортирует кадры или сохраняет последние минуты из фонового буфера.

Обязательные функциональные возможности:

- запись всего экрана;
- запись выбранного монитора;
- запись выделенной области экрана;
- захват конкретного окна или приложения без лишнего содержимого вокруг;
- запись микрофона;
- запись системного звука;
- основной результат записи сохраняется как локальный видеофайл, а не как каталог кадров;
- указатель мыши виден в записанном видео, если пользователь явно не отключил его;
- фильтрация системного звука по приложениям там, где платформа это позволяет;
- фоновая запись с сохранением последних `N` минут по кнопке или горячей клавише;
- раскадровка создается только отдельным действием после записи;
- раскадровка экспортируется либо как набор отдельных PNG, либо как один PNG-контактный лист;
- CLI-режим для автоматизации;
- Compose Multiplatform GUI для повседневного управления;
- поддержка Windows, macOS, Linux Ubuntu и Android.

Продуктовые принципы:

- локальная обработка без обязательного облака;
- явное управление разрешениями на экран, микрофон и системный звук;
- понятное хранение файлов и настроек;
- отсутствие скрытых платных ограничений на базовую запись;
- открытая архитектура, которую можно проверять, тестировать и расширять;
- практичный интерфейс без перегруженности.

Технические ограничения:

- основной язык - Kotlin;
- сборка - Gradle Wrapper;
- UI - Compose Multiplatform;
- асинхронность - Kotlin Coroutines и Flow;
- бизнес-логика должна быть платформонезависимой;
- непрерывные состояния должны публиковаться через `StateFlow` или `SharedFlow`;
- запрещено использовать `GlobalScope`;
- запрещено хардкодить `Dispatchers.IO`, `Dispatchers.Default` и `Dispatchers.Main` в бизнес-логике;
- диспетчеры и `CoroutineScope` должны передаваться через конструкторы и связываться на границах приложения;
- запрещено использовать `Thread.sleep`;
- native/device API должны быть спрятаны за небольшими platform adapter слоями;
- для JVM/Android использовать JNI или платформенные JVM/Android API;
- для Kotlin/Native использовать `cinterop`;
- тесты писать через `kotlin.test` и `kotlinx-coroutines-test`;
- реальные устройства в unit-тестах заменять fake adapters.

Целевое состояние проекта:

- есть модульная архитектура с отдельными слоями domain/core, platform adapters, audio, encoding, settings, CLI и GUI;
- запись можно запускать из CLI;
- основные сценарии можно выполнять из Compose GUI;
- pipeline записи корректно обрабатывает start, pause, stop, cancel, failure и cleanup;
- аудио и видео синхронизируются по единой модели времени;
- настройки валидируются до старта записи;
- платформенные различия изолированы в adapter-модулях;
- сборка, тесты и базовая проверка проходят через Gradle Wrapper;
- каждый крупный сценарий имеет Definition of Done и тестовое покрытие на уровне доменной логики.
